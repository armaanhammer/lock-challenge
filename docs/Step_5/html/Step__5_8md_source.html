304 
305 * Pass:
306  * Avoid anomalous behavior, including but not limited to:
307  * unhandled exceptions
308  * seg faults
309  * other memory errors
310  * Produce output identical to examples from prompt.
311  * Successfully add commands to map.
312  * If conflict, throw and handle exception.
313  * Parse strings for JSON.
314  * If no JSON present, throw and handle exception.
315  * If JSON present, parse JSON for "command" and "payload" key/value pairs.
316  * If one or both not present, throw and handle exception.
317  * If both present, query map for matching command.
318  * If no matching command, throw and handle exception.
319  * If matching command found, pass value stored by "payload" key to command handler function.
320  * Inside of each command handler function, parse value passed in for JSON.
321  * If no match for required key/value pair, throw and handle exception.
322  * If match for required key/value pair, safely determine if value contains valid data.
323  * If value does not contain valid data, throw and handle exception.
324  * If value either contains only valid data, or both valid and invalid data, handle command using valid data.
325  * Continue indefinitely until "exit" command successfully handled.
326 * Fail: Any other result.

335 ## Design Constraints
336 
337 The project involved gluing together parts of an already-established codebase, and creating new functionality on top of it. To facilitate smooth operation, needed to establish what constraints the existing code placed on my implementations. Here are the constraints identified:
338 
339 * main()
340  * adds command handlers in Controller class to CommandDispatcher using addCommandHandler
341  * invokes CommandDispatcher::dispatchCommand using a raw string.
342 * CommandDispatcher()
343  * stores command handlers in Controller class using a private map
344  * invokes command handler functions using either rapidjson::value or rapidjson::document objects
345 * Controller()
346  * implements command handler functions
347  * return values are bool
348 
349 ## Design Decisions
350 
351 To design around these constraints, I needed to parse out a reasonable division of labor for the functions I implemented, and determine how information would be passed back and forth. I chose to allow functions at all depths to print to the user, except on error conditions. On errors, I chose to throw exceptions which get passed back to and handled by main(). Were I to continue to improve on this code, I would consider creating a shared print function for all the other functions to call so the output stream is consolidated to one place.
352 
353 Here is a psuedo-code division of labor for my implementations:
354 
355 * main()
356  * prints success to user
357  * catches failure message exceptions from all other parts of program and passes them to exceptionPrinter
358 * exceptionPrinter()
359  * prints to user
360  * returns to main
361 * CommandDispatcher()
362  * parses raw string from main into a rapidjson::document
363  * performs initial checks to verify that "command" and "payload" values exist
364  * attempts to find command handler function matching "command"
365  * extracts "payload" value from document and passes by reference to command handler
366  * prints success to user
367  * passes failures as exceptions back to main
368  * passes exceptions from Controller back to main
369 * Controller()
370  * performs commanded functions
371  * functions check whether their individual parameters are satisfied.
372  * prints success to user
373  * passes failures as exceptions back to CommandDispatcher
